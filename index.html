O que venho aprendendo

Entre vários assuntos que já vimos, preciso destacar dois que me entusiasmaram bastante: Bash e Git/GitHub.

Eu sabia que essas ferramentas eram muito importantes pra um desenvolvedor, mas achava que eram bem dificeis de aprender fora de um contexto em que fossem usadas rotineiramente (e eu tava certo).

A Trybe além de ensinar o funcionamento, criou uma rotina que tem permitido essaa prática.
Não imagino uma forma melhor de aprender.

Quebrei mesmo a cabeça com Git

No segundo dia de Git/GitHub eu fiquei perdido com a quantidade de comandos, apesar dos conceitos estarem relativamente claros.

Então criei uma planilha com cada comando e variação, seguido de uma descrição. Ajudou a memorizar e agora tenho uma tabela pra consultar.

Comando Opções Descrição
git clone URL_do_repositório Importa o repositório Git cujo link foi passado como parâmetro
git branch nome_da_branch Cria uma nova branch a partir da atual
git checkout nome_da_branch Sai da branch atual e vai para a indicada
git checkout SHA_do_commit Vai para a versão da branch do commit informado
git checkout -b nome_da_branch Cria uma branch e faz checkout pra ela
git status Verifica se existem modificações p/ adicionar ou commitar
git add . Adiciona todas as modificações à Staging Area
git add nome_do_arquivo Adiciona um arquivo específico à Staging Area
git commit -m "mensagem_desejada" Commita as modificações que estão na Staging Area
git push -u origin nome_da_branch Faz o primeiro push de uma branch para o repositório remoto (se houver)
